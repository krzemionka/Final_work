A tutorial on how to build API documentation in Postman

This tutorial consists of 4 simple steps:

  1. Install Postman.
  2. Create a new project.
  3. Add resources to the project.
  4. Do exercises.
  5. Build API documentation.

 

Step 1: Install Postman

The first step is to install the Postman tool, which is a platform for testing and documenting APIs. You can download Postman from the official site at Download Postman | Get Started for Free and install it on your computer.

Postman starting page

Step 2: Create a new project

To create a new project select the New button on the navigation bar and select API Documentation. Postman will take you to Create a new collection button. Click on the orange button Next and name your collection (give a name to your API). In the following example, the collection is named “API”:

Creating a new collection in Postman

 

Step 3: Add resources to the project

The screen is divided into three sections: My Workspace with navigation bar and sidebars, Overview which is the main content area for personal work and Documentation. Add a request in My Workspace. Postman will create a New request with three with empty fields:

  • New Collection
  • HTTP request method
  • URL field

Creating a New request in Postman

In the New Collection write Get users, in the HTTP request method, select GET, and in the URL field insert https://api.example.com/users. You may also fill in a Description field to write short and precise information on what an endpoint does, e.g. Create a new user, Remove a user with the following number. Your first endpoint Get Users is ready.

Endpoint Get Users in Postman

Add another request by selecting API project and choosing More options. Select Add requests and make other endpoints. Use Key and Value in Query Params. Below you will have other endpoints to create.

Endpoint 2: Get user information

New Collection: Get User by ID

HTTP request method: GET

URL field: https://api.example.com/users/{id}

Path Parameter: Key (type: string), Value: 123)

 

Endpoint 3: Add a new user

New Collection: Create User

HTTP request method: POST

URL field: https://api.example.com/users

Path Parameter: Key (type: string), Value: 123

Header: Content-Type: application/json

Body: JSON

 

Endpoint 4: Update user information

New Collection : Update Use

HTTP request method: PUT

URL field: https://api.example.com/users/{id}

Path Parameter: Key (type: string), Value: 123

Header: Content-Type: application/json

Body: JSON

At the end you will have a Postman collection.

Postman collection

 

Step 4: Generate your API documentation

Creating this simple API was necessary to see how our API documentation would look like. Once again, select API project and choose More options. Select View documentation. Postman will show you a nicely looking, neat and ordered documentation. You may also publish API documentation on the Postman platform. Follow the steps:

  • Choose the project name in the Navigation panel and select Publish Documentation.
  • Select Postman as the documentation format.
  • Enter the name of the documentation, select the Publishing options and choose Publish.
  • View the API documentation.

Once you publish your API documentation, you will receive a unique link that you can share with others. Open this link in your web browser to view the generated API documentation in Postman.
