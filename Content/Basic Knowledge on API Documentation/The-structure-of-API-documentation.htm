The structure of API documentation

API documentation should include a certain number of information so as to increase clarity and usability.

  • Introduction/overview

The overview should contain a summary of the API and the problem it is solving. It could also include the benefits of using this particular API over other similar APIs. See the example below.

source: Bootstrap

 

 

 

  • Tutorials and case study

Tutorials are the main part of the API - they teach users the concept of the API and how to use it effectively. It can also include links for reference and step-by-step guides how to integrate the API and what proper functioning looks like.

source: https://code.visualstudio.com/

 

  • Authentication and authorization

APIs handle thousands of requests and enormous amounts of data so authentication is about protecting data from hackers. Most API docs start with authentication and authorization. API Authentication verifies a user's identity and gives them access to resources, authorization, on the other hand is about verifying whether the user is allowed to perform what they are trying to do in the system. Also, the API documentation should explain authentication requests, possible errors, token expiration times and an explanation on authentication sensitivity (reminders that keys can’t be shared and where they can be used).

  • Endpoints, URI Patterns, and HTTP Methods

API endpoint definitions are the point at which the API connects with the software program. The endpoints only show the end of the path, hence their name. They show access to the resource and HTTP methods the endpoint interacts with, namely GET, POST, or DELETE.

  • Request and Response Formats

You must document the request and response formats so the user knows what to expect. The request is a URL from a client asking for a resource, while the response is feedback from the server.

  • Error Codes and Error Handling

Expected error codes in documentation to help users understand the errors so as to increase the usability of the API documentation; examples of calls, errors, responses and other operations that a developer interacts with the API. Make sure that your examples are relevant to the user's needs and provide enough detail to be helpful.

source: www.oracle.com

  • Sample Code Snippets

They show users how to integrate the API in various languages and formats. Include how to install and integrate SDKs (software development kits) in various languages in the documentation.

  • Examples

When users understand how the API works, it’s good to give them examples that show successful examples of calls, responses, error handling and other operations that they might encounter when using the API.

  • API Versioning and Change Logs

It ensures the delivery of uninterrupted services to your users. Versioning can enhance the API with new versions without affecting client applications.
